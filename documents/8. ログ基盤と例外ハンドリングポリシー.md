# ログ基盤（Serilog）と例外ハンドリングポリシー

## 1. Serilog 設定

### パッケージ構成

| パッケージ                   | 用途           |
| ---------------------------- | -------------- |
| `Serilog`                    | コアライブラリ |
| `Serilog.Sinks.File`         | ファイル出力   |
| `Serilog.Extensions.Logging` | `ILogger` 統合 |

### ログレベル方針

| レベル          | 用途                                   | 出力先                  |
| --------------- | -------------------------------------- | ----------------------- |
| **Fatal**       | アプリケーション停止級のエラー         | ファイル                |
| **Error**       | 予期しない例外・操作失敗               | ファイル                |
| **Warning**     | 注意喚起（リトライ可能な失敗等）       | ファイル                |
| **Information** | 一般情報（起動、画面遷移、設定保存等） | ファイル                |
| **Debug**       | 開発時デバッグ情報                     | ファイル + デバッグ出力 |
| **Verbose**     | 詳細トレース                           | 出力しない              |

### 出力先

| 出力先       | パス                                                   | ローテーション   |
| ------------ | ------------------------------------------------------ | ---------------- |
| ファイル     | `%LocalAppData%/AmeCapture/logs/amecapture-{Date}.log` | 日次、30日保持   |
| デバッグ出力 | Visual Studio / デバッガ                               | DEBUG ビルドのみ |

### 出力フォーマット

```
2026-04-20 12:34:56.789 +09:00 [INF] Application started
2026-04-20 12:34:57.123 +09:00 [ERR] Failed to save image
System.IO.IOException: Disk full
```

### フィルタリング

- `Microsoft.*` → Warning 以上
- `System.*` → Warning 以上
- アプリコード → Information 以上

## 2. 例外ハンドリングポリシー

### 未処理例外のキャッチ

| ハンドラ                                     | 対象                       | アクション                                    |
| -------------------------------------------- | -------------------------- | --------------------------------------------- |
| `AppDomain.CurrentDomain.UnhandledException` | メインスレッドの未処理例外 | Fatal ログ出力 → アプリ終了                   |
| `TaskScheduler.UnobservedTaskException`      | 非同期タスクの未処理例外   | Error ログ出力 → タスクを観測済みとしてマーク |

### 各レイヤーの例外処理方針

| レイヤー           | 方針                                                           |
| ------------------ | -------------------------------------------------------------- |
| **Domain**         | 例外をスローしない（バリデーションエラーは `Result` 型で返す） |
| **Application**    | ビジネスロジック例外をキャッチ → ログ出力 → ViewModel に通知   |
| **Infrastructure** | 外部 I/O 例外をキャッチ → ログ出力 → Application 層に伝播      |
| **App (Views)**    | UI スレッド例外をキャッチ → ユーザーに通知（Alert） + ログ出力 |

### 実装箇所

- **`MauiProgram.cs`**: グローバル例外ハンドラ登録
- **ViewModel**: `try-catch` でサービス呼び出しをラップ、エラー時に `Log.Error()` + UI通知
- **Repository**: `try-catch` でDB例外をキャッチ、`Log.Warning()` / `Log.Error()`
