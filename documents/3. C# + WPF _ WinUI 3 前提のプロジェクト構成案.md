# 3. C# + WPF / WinUI 3 前提のプロジェクト構成案

ここは結論から言うと、**最初は WPF 推奨**です。

## 3.1 なぜ最初は WPF 推奨か

このアプリで必要になるのは以下です。

- オーバーレイ表示
- グローバルホットキー
- タスクトレイ
- ネイティブAPI連携
- ウィンドウ制御
- キャプチャUIの細かい制御

こういう **Windowsデスクトップ感の強いアプリ** は、\
最初のMVPなら **WPFの方が安定して進めやすい** です。

### おすすめ方針

- **MVP〜v1:** WPF
- **将来的な再構築候補:** WinUI 3

## 3.6 技術選定メモ

### UI

- **WPF**
  - MVP向き
  - 枯れている
  - カスタム描画しやすい

### 画像処理

- **SkiaSharp** 推奨
  - 注釈や加工の自由度が高い

### DB

- **SQLite**
  - ローカル完結に最適

### ログ

- `Serilog` などの採用候補

### 動画

- 初期は後回し
- 入れるなら `FFmpeg` 連携 or Windows Capture API

## 3.7 MVPでの保存ポリシー案

### 画像

- 原本は `originals/`
- 編集済みは `edited/`
- サムネイルは `thumbnails/`

### 保存方式

初期は **非破壊保存を推奨**

- 元画像を残す
- 編集後は別ファイルで保存
- DBに current\_path を持たせる

これにすると後から再編集や履歴追加がしやすいです。
