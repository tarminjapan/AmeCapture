# 1. GitHub Issues / Backlog 形式に落とした開発計画

以下はそのまま Issue 化しやすいように、\
**Epic → Story → Task → 完了条件（Acceptance Criteria）** の形式で整理しています。

## Milestone 構成案

- **M1: Foundation**
- **M2: Workspace MVP**
- **M3: Capture MVP**
- **M4: Editor MVP**
- **M5: UX Enhancement**
- **M6: Video Capture**
- **M7: Advanced Features**

## Epic 0: Foundation（基盤構築） - [DONE]

### Issue: プロジェクトの初期セットアップを行う - [DONE]

**ラベル:** `epic:foundation`, `priority:high`, `type:setup`

#### 内容

- Cargo workspace作成 / Tauriプロジェクト初期化
- クレート分割（Rustバックエンド / Reactフロントエンド）
- DIパターン導入（trait + 実装の注入）
- Logging導入（`tracing` / `log` クレート）
- 設定ファイル管理（Tauriの設定API活用）
- SQLite接続準備（`rusqlite` または `sqlx`）
- 共通エラーハンドリング（`anyhow` / `thiserror`）

#### Acceptance Criteria

- Tauriアプリが起動する
- Tauri State管理でServiceが注入・利用できる
- 設定ファイルを読み込める
- SQLite DBファイルを初期化できる
- ログファイルが出力される

### Issue: SQLiteスキーマを定義する - [DONE]

**ラベル:** `epic:foundation`, `priority:high`, `type:data`

#### 内容

テーブル候補:

- `workspace_items`
- `tags`
- `workspace_item_tags`
- `app_settings`
- `capture_presets`
- `recent_actions`

#### Acceptance Criteria

- 初回起動時にDBが自動作成される
- テーブル作成が完了する
- 最低限 `workspace_items` に対してCRUDできる

### Issue: ファイル保存構造を決定・実装する - [DONE]

**ラベル:** `epic:foundation`, `priority:high`, `type:storage`

#### 内容

保存先例:

```text
%AppData%/Local/MyCaptureApp/
 ├─ data/
 │   ├─ originals/
 │   ├─ edited/
 │   ├─ thumbnails/
 │   └─ videos/
 ├─ db/
 ├─ logs/
 └─ settings/
```

#### Acceptance Criteria

- 保存先ルートが自動生成される
- originals / edited / thumbnails / videos が作成される
- DBメタデータと実ファイルパスが整合する

## Epic 1: Workspace Core（最優先）

### Issue: ワークスペース一覧画面を作成する

**ラベル:** `epic:workspace`, `priority:highest`, `type:feature`

#### 内容

- サムネイル一覧

- 種別（画像/動画）表示

#### Acceptance Criteria

- DBにあるアイテムが一覧表示される

- 一覧から1件選択できる

### Issue: ワークスペース詳細ペインを作成する

**ラベル:** `epic:workspace`, `priority:high`, `type:feature`

#### 内容

- プレビュー
- タイトル

- ファイルパス
- タグ
- 操作ボタン

#### Acceptance Criteria

- 一覧で選択したアイテムの詳細が表示される
- タイトル変更が可能
- 保存先フォルダを開ける

### Issue: ワークスペース内で削除・名前変更を実装する

#### Acceptance Criteria

### Issue: タグ付け機能を実装する

**ラベル:** `epic:workspace`, `priority:medium`, `type:feature`

#### Acceptance Criteria

## Epic 2: Capture Core（MVPキャプチャ）

### Issue: 全画面キャプチャを実装する

**ラベル:** `epic:capture`, `priority:highest`, `type:feature`

#### Acceptance Criteria

- 全画面のスクリーンショットが保存される
- 保存後にワークスペースへ登録される
- サムネイルが生成される

### Issue: 指定領域キャプチャを実装する

**ラベル:** `epic:capture`, `priority:highest`, `type:feature`

#### 内容

- 半透明オーバーレイ
- ドラッグ選択
- 範囲確定
- Escキャンセル

#### Acceptance Criteria

- ユーザーが範囲をドラッグ選択できる
- 範囲が画像として保存される
- キャンセル時に何も保存されない

### Issue: ウィンドウキャプチャを実装する

**ラベル:** `epic:capture`, `priority:high`, `type:feature`

#### Acceptance Criteria

- ウィンドウ単位でキャプチャできる
- 複数ウィンドウがある状態でも対象を判別できる
- キャプチャ画像が保存・登録される

### Issue: グローバルホットキーを実装する

**ラベル:** `epic:capture`, `priority:highest`, `type:feature`

#### 例

- `PrintScreen`: 範囲キャプチャ
- `Ctrl + PrintScreen`: 全画面
- `Alt + PrintScreen`: ウィンドウ

#### Acceptance Criteria

- アプリ非アクティブ時でもホットキーが動作する
- 競合時に設定変更できる余地がある
- ホットキーからキャプチャ起動できる

### Issue: タスクトレイ常駐を実装する

**ラベル:** `epic:capture`, `priority:high`, `type:feature`

#### Acceptance Criteria

- 最小化でトレイ格納できる
- トレイメニューからキャプチャ起動できる
- 終了操作が可能

### Issue: クリップボードコピーを実装する

**ラベル:** `epic:capture`, `priority:high`, `type:feature`

#### Acceptance Criteria

- キャプチャ直後に画像をコピーできる
- ワークスペースから再コピーできる
- Office/Teams等へ貼り付け可能
