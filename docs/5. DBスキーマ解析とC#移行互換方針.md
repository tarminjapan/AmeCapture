# 既存DBスキーマ解析とC#移行互換方針

## 1. 既存スキーマ一覧

SQLite スキーマは `src/AmeCapture.Infrastructure/Database/DatabaseInitializer.cs` で定義されている。

### テーブル一覧

| テーブル名            | 用途                                  |
| --------------------- | ------------------------------------- |
| `workspace_items`     | キャプチャ画像・動画のメタデータ      |
| `tags`                | タグ名                                |
| `workspace_item_tags` | アイテム⇔タグの中間テーブル（多対多） |
| `app_settings`        | アプリ設定（キー・バリュー）          |

### workspace_items

| カラム           | 型      | 制約                     | 説明                         |
| ---------------- | ------- | ------------------------ | ---------------------------- |
| `id`             | TEXT    | PRIMARY KEY              | UUID                         |
| `type`           | TEXT    | NOT NULL DEFAULT 'image' | "image" or "video"           |
| `original_path`  | TEXT    | NOT NULL                 | オリジナルファイルパス       |
| `current_path`   | TEXT    | NOT NULL                 | 現在のファイルパス（編集後） |
| `thumbnail_path` | TEXT    |                          | サムネイルパス               |
| `title`          | TEXT    | NOT NULL                 | 表示タイトル                 |
| `created_at`     | TEXT    | NOT NULL                 | 作成日時 (ISO 8601)          |
| `updated_at`     | TEXT    | NOT NULL                 | 更新日時 (ISO 8601)          |
| `is_favorite`    | INTEGER | NOT NULL DEFAULT 0       | お気に入りフラグ (0/1)       |
| `metadata_json`  | TEXT    |                          | 追加メタデータ (JSON)        |

### tags

| カラム | 型   | 制約            | 説明   |
| ------ | ---- | --------------- | ------ |
| `id`   | TEXT | PRIMARY KEY     | UUID   |
| `name` | TEXT | NOT NULL UNIQUE | タグ名 |

### workspace_item_tags

| カラム              | 型   | 制約                                       | 説明       |
| ------------------- | ---- | ------------------------------------------ | ---------- |
| `workspace_item_id` | TEXT | NOT NULL, FK → workspace_items(id) CASCADE | アイテムID |
| `tag_id`            | TEXT | NOT NULL, FK → tags(id) CASCADE            | タグID     |

PK: `(workspace_item_id, tag_id)`

### app_settings

| カラム  | 型   | 制約        | 説明     |
| ------- | ---- | ----------- | -------- |
| `key`   | TEXT | PRIMARY KEY | 設定キー |
| `value` | TEXT | NOT NULL    | 設定値   |

### インデックス

| インデックス名                    | 対象                               |
| --------------------------------- | ---------------------------------- |
| `idx_workspace_items_created_at`  | `workspace_items(created_at DESC)` |
| `idx_workspace_items_is_favorite` | `workspace_items(is_favorite)`     |

### SQLite PRAGMA

- `journal_mode=WAL`
- `foreign_keys=ON`

## 2. 互換方針

### 基本方針: **スキーマ互換維持（スキーマ変更なし）**

C# 移行後も同じ `amecapture.db` を読み書きできるよう、**テーブル定義・カラム名・データ形式を一切変更しない。**

### 移行方針詳細

| 項目             | 方針                                                        |
| ---------------- | ----------------------------------------------------------- |
| マイグレーション | C# 側でも `CREATE TABLE IF NOT EXISTS` で同定義を維持       |
| 型マッピング     | `INTEGER` → `bool`/`int`, `TEXT` → `string`/`DateTime`      |
| 日時形式         | 常に UTC で保存（`DateTime.UtcNow.ToString("o")`）          |
| UUID             | `Guid.NewGuid().ToString()` で生成                          |
| 外部キー         | 接続文字列の `Foreign Keys=True` または `PRAGMA` で有効化   |
| WALモード        | `PRAGMA journal_mode=WAL` を維持                            |
| ファイルパス     | アプリデータディレクトリ内の `amecapture.db` をそのまま使用 |

### C# 型マッピング

```csharp
// workspace_items
public class WorkspaceItem
{
    public string Id { get; set; }                    // TEXT PK
    public WorkspaceItemType ItemType { get; set; }   // TEXT ("image"/"video", 小文字で保存)
    public string OriginalPath { get; set; }          // TEXT
    public string CurrentPath { get; set; }           // TEXT
    public string? ThumbnailPath { get; set; }        // TEXT (nullable)
    public string Title { get; set; }                 // TEXT
    public DateTime CreatedAt { get; set; }           // TEXT (ISO 8601)
    public DateTime UpdatedAt { get; set; }           // TEXT (ISO 8601)
    public bool IsFavorite { get; set; }              // INTEGER (0/1)
    public string? MetadataJson { get; set; }         // TEXT (nullable)
}
```

### データアクセスライブラリ

- **Microsoft.Data.Sqlite** (ADO.NET ベース、軽量)
- ORM は導入せず、生 SQL + 手動マッピングで同一クエリを維持

## 3. SQLite 接続テスト

C# 側のインフラストラクチャ層で SQLite 接続を確認するテストを実装済み（`AmeCapture.Tests` プロジェクト内）。

接続確認手順:

1. `Microsoft.Data.Sqlite` でインメモリ DB を開く
2. `PRAGMA foreign_keys=ON` を実行
3. マイグレーション SQL を実行
4. テーブル存在確認
