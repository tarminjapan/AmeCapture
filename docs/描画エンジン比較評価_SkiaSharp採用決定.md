# 描画エンジン比較評価: MAUI GraphicsView vs SkiaSharp

## 比較目的

AmeCapture の画像編集機能（矢印、矩形、モザイク、テキスト注釈など）における描画エンジンとして、MAUI GraphicsView と SkiaSharp のどちらを採用するか評価する。

## 評価基準

| 基準                       | 重要度 | 説明                                       |
| -------------------------- | ------ | ------------------------------------------ |
| 描画パフォーマンス         | 高     | 大画像（4K以上）での操作遅延               |
| 画像操作能力               | 高     | ピクセルレベル操作、モザイク処理、クロップ |
| テキスト描画品質           | 中     | 注釈テキストのレンダリング品質             |
| クロスプラットフォーム対応 | 中     | Windowsメインだが将来の拡張性              |
| API 豊富さ                 | 中     | 図形描画、パス操作、画像フィルタ           |
| コミュニティ・ドキュメント | 低     | 情報の入手しやすさ                         |

## MAUI GraphicsView の評価

### 利点

- MAUI に標準搭載、追加パッケージ不要
- `IDrawable` インターフェースによるシンプルな描画モデル
- `Microsoft.Maui.Graphics` による基本的な図形描画

### 欠点

- ピクセルレベルの画像操作が困難（モザイク処理に不向き）
- 画像フィルタ機能が限定されている
- 高度なパス操作のサポートが不十分
- `ICanvas` API の機能セットが限定的
- テキスト描画のカスタマイズ性が低い
- 大画像での再描画時にパフォーマンス問題が報告されている

### POC 結果

- 矩形・矢印の描画: 可能だが線幅やアンチエイリアス制御に制限
- モザイク処理: ピクセルアクセス API がないため実装不可
- テキスト描画: 基本的な描画は可能だがフォント制御が限定的

## SkiaSharp の評価

### 利点

- Google Skia エンジンベースの高性能 2D グラフィックス
- `SKBitmap` によるピクセルレベル操作が可能
- 豊富な画像フィルタ（ぼかし、モザイク、色変換など）
- 高度なパス操作とベジェ曲線サポート
- テキスト描画の高品質レンダリング
- `SKCanvasView` による MAUI 統合が容易
- 大画像でも安定したパフォーマンス
- HarfBuzz によるテキストシェーピング対応

### 欠点

- 追加 NuGet パッケージが必要（~15MB）
- ネイティブライブラリの依存関係
- 学習コストがやや高い

### POC 結果

- 矩形・矢印の描画: `SKCanvas.DrawLine`, `SKPath` で高品質に描画可能
- モザイク処理: `SKBitmap` のピクセルアクセスで実装可能
- テキスト描画: `SKCanvas.DrawText` でフォント・サイズ・色を完全制御
- クロップ: `SKBitmap.ExtractSubset` で効率的に処理
- 大画像(3840x2160): 描画操作が 60fps でスムーズに動作

## 比較サマリ

| 基準                   | GraphicsView | SkiaSharp |
| ---------------------- | ------------ | --------- |
| 基本図形描画           | ○            | ◎         |
| ピクセル操作           | ×            | ◎         |
| モザイク処理           | ×            | ◎         |
| テキスト描画           | △            | ◎         |
| 大画像パフォーマンス   | △            | ◎         |
| パッケージサイズ       | ◎（内蔵）    | △（追加） |
| API 豊富さ             | △            | ◎         |
| クロスプラットフォーム | ◎            | ◎         |

## 決定: SkiaSharp を採用

### 理由

1. **モザイク処理が必須要件**: GraphicsView ではピクセルレベルの操作が不可能であり、モザイク注釈を実装できない
2. **画像保存時のレンダリング一貫性**: インタラクティブ描画と画像保存時のレンダリングに同じエンジンを使用することで、WYSIWYG を保証
3. **パフォーマンス**: SkiaSharp はネイティブ C++ エンジンベースであり、大画像での操作において MAUI GraphicsView より高速
4. **将来の拡張性**: ぼかし、ハイライト、透過などの編集機能追加が容易

### 使用パッケージ

- `SkiaSharp`: コア描画ライブラリ（Infrastructure 層）
- `SkiaSharp.Views.Maui.Controls`: MAUI 統合コントロール（App 層）

### アーキテクチャ

- **Infrastructure 層**: `SkiaSharpEditorService` が画像への注釈適用を担当
- **App 層**: `SKCanvasView` ベースのエディタページがインタラクティブ描画を担当
