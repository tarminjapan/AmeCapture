# 2. AI Agentに渡すための要件定義書テンプレート（＋今回の案件で埋めた版）

以下は **そのまま開発開始に使える要件定義書** です。\
テンプレートとしても再利用できるよう、構造を明確にしています。

## 要件定義書

**プロジェクト名（仮）:** LocalCapture Workspace\
**対象OS:** Windows 10/11\
**バックエンド言語:** C#\
**UIフレームワーク:** .NET MAUI\
**DB:** SQLite\
**保存方式:** ローカルファイル + SQLiteメタデータ\
**クラウド連携:** 対象外（実装しない）

## 1. 目的

業務で使いやすい、**ローカル完結型のスクリーンキャプチャアプリ** を構築する。\
本アプリは以下を重視する。

- キャプチャのしやすさ
- 撮影した画像/動画の整理・再利用のしやすさ
- 注釈編集のしやすさ
- 企業利用しやすいローカル完結構成
- クラウドなし・外部送信なし

## 2. 背景

既存のスクリーンキャプチャツールでは、クラウド機能や外部連携機能がセキュリティ・運用ポリシー上の制約になる場合がある。\
そのため、**クラウド連携を持たず、業務利用に特化した代替アプリ** を自社開発/個人開発する。

## 3. スコープ

### 3.1 初期スコープ（MVP）

- 指定領域キャプチャ
- 全画面キャプチャ
- ウィンドウキャプチャ
- ワークスペース（履歴一覧）
- サムネイル生成
- クリップボードコピー
- 削除 / 名前変更
- タスクトレイ常駐
- グローバルホットキー
- 基本編集（矢印、テキスト、四角形、モザイク、トリミング）
- 保存
- Undo / Redo

### 3.2 将来スコープ

- 動画録画
- 遅延キャプチャ
- 前回領域再撮影
- 保存形式変換
- OCR
- スクロールキャプチャ
- 外部エディタ連携

### 3.3 明確に対象外

- クラウド同期
- クラウド共有
- Webアカウント
- チーム共有機能
- SaaS化
- 外部サーバー依存
- テレメトリ送信（デフォルトOFF or 非搭載）

## 4. 想定ユーザー

- ITエンジニア
- 業務アプリ開発者
- テスト担当者
- 仕様書/障害報告/手順書を作成する人
- 社内でセキュリティ制約の強い環境にいる人

## 5. 主要ユースケース

### UC-01: 範囲キャプチャ

ユーザーはホットキーを押し、任意範囲をドラッグしてスクリーンショットを保存する。

### UC-02: ワークスペースで再利用

ユーザーは撮影済み画像を一覧から探し、コピーまたは編集する。

### UC-03: バグ報告用に注釈を入れる

ユーザーは矢印・テキスト・モザイクを追加し、編集済み画像を保存する。

### UC-04: 不要なキャプチャを整理する

ユーザーは履歴から削除または名前変更する。

### UC-05: 何度も同じ範囲を再撮影する（将来）

ユーザーは前回と同じ領域を使って連続的に資料を作成する。

## 7. 非機能要件

### NFR-01: セキュリティ

- 外部サーバーへ送信しないこと
- オフライン動作可能であること
- ローカル保存のみで完結すること

### NFR-02: 性能

- 通常キャプチャはユーザー操作後 1 秒以内に保存完了を目指す
- ワークスペース一覧表示は 1000件規模でも実用速度を維持する

### NFR-03: 可用性

- アプリ起動時に破損データがあっても全体起動不能にならないこと
- サムネイル生成失敗時も原本は保持すること

### NFR-04: 拡張性

- 機能追加しやすいプロジェクト構成であること
- 画像編集、録画、OCRを後付けしやすいこと

### NFR-05: 保守性

- C# バックエンド（Infrastructure層）とMAUIフロントエンド（App層）で責務分離すること
- Domain / Application / Infrastructure / App レイヤーを分離すること
- フロントエンドは MVVM パターンで状態管理すること
- ログとエラーハンドリングを共通化すること

## 9. 制約条件

- Windows専用
- クラウド・サーバー依存禁止
- MVPではスクロールキャプチャ対象外
- MVPではOCR対象外
- MVPでは動画編集対象外

## 10. 成功条件

MVP成功条件は以下。

1. ホットキーから範囲キャプチャできる
2. 画像が自動保存される
3. ワークスペースで一覧表示される
4. 矢印・テキスト・モザイクを付けて保存できる
5. クリップボードへコピーできる
6. 企業内のローカル環境で完結利用できる
